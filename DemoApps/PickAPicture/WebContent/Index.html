Introduction

Internet used to be a collection of content, which was going well with each site isolated. With the evolution of the Internet, applications become more important than before. And the connection between them become closer and closer.

There are more and more people to care about this problem. Some applications provide open API for third part applications to get involved with them, and they are ready to be as a component of the Internet. However, it's hard and complicated for programmers to develop such third-part application.

We present an online system with a communication protocol based on it. It could make the applications on the Internet to be connected with each other. What's more, users can customize their self Internet linkages through it.

InternetRuntime integrated the Internet services and applications. It makes the developer's job simple.

API

The InternetRuntime is easy to get access into. What developers need to do is to finish a configuration file. The function of this file is to install the service or application to out system InternetRuntime. The client side just register the match the signal produced and the signal receiver. Then our system will do all left work. It organized all the signal and the applications just like a Unix operate system.

Signal Registration(Install an application)

How to install an application in our InternetRuntime
<Application>
<Name>PickPic<Name/>
<Listeners>
  <HttpListner>
    <Url>http://localhost:8080/PickAPicture/PickAPicture<Url/>
    <MatchRule type="urlregex">.*/signal/getPicture.*<MatchRule/>
  <HttpListener/>
  <ClientListner>
    <ClientType>osclient://internetos/clientsignal/scriptloader<ClientType/>
    <MatchRule type="urlregex">.*/signal/selectPicture.**<MatchRule/>
  <ClientListner/>
<Listeners/>
<Application/>
Application
Here,developers can install an application into our InternetRuntime system.
Name
The unique name of an application.
Listeners
Developers state the signal, the listener of signal and the type of listener in this area.
HttpListener
Optional, if the signal is http signal, developers can customize the match rule of the signal and the listener here.
Url
This is one side of signal-producer and signal-receiver. If it is a web page, it must state in the url label
MatchRule
Required, in this label, the specific signai is defined.
ClientListner
Optional, if the signal is client signal, developers can customize the match rule of the signal and the listener here.

Invocation

Next, invoke this jsp configuration file in a firefox browser that has installed the client of our InternetRuntime. The installation of the application is successful.
